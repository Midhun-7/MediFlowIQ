# 1. Project Foundation

## Project Name

**MediFlowIQ**

## Core Purpose

MediFlowIQ is a centralized hospital coordination system designed to optimize patient flow and emergency response. It provides real-time queue management, ambulance tracking, and intelligent prioritization to reduce waiting times and improve operational efficiency.

The system is built using a **simulation-first approach**, using anonymized patient data and simulated GPS streams to replicate real-world hospital environments while respecting privacy constraints.

---

## Tech Stack

### Backend

* **Java** – core backend language for building scalable services
* **Spring Boot** – framework for building REST APIs and real-time backend systems

### Frontend

* **TypeScript + React + Tailwind** – dashboard and user interface

### Real-Time Communication

* **WebSockets (Socket.io)** – live updates for queue and ambulance tracking

### Maps & Location

* **Leaflet.js / Google Maps API** – map visualization
* Simulated GPS data (Phase 1)

### Database

* **PostgreSQL** – structured data (patients, queues)
* **Redis (optional)** – real-time queue state

# 3. Architecture & Testing

## Architecture Pattern

* **Modular Monolith (initial phase)**
  → Enables fast iteration and easier debugging

* Designed for future transition to:

  * Microservices (queue service, tracking service, auth service)

---

## Core Components

### 1. Queue Management Service

* Token-based system
* Priority handling:

  * Emergency > High-risk > Normal
* Estimated waiting time calculation

---

### 2. Ambulance Tracking Service

* Simulated GPS movement (Phase 1)
* Real-time updates via WebSockets
* Displays ambulance location and ETA

---

### 3. API Layer

* REST APIs for system interaction
* Handles authentication and data access

---

### 4. Frontend Dashboard

* Live queue visualization
* Map-based ambulance tracking
* Admin controls

---

### 5. Security Layer

* Authentication (JWT-based)
* Role-based access:

  * Admin
  * Doctor
  * Staff
* Simulated/anonymized patient data

# 6. Design Philosophy

* Build with **simulation first**, real-world later
* Focus on **real-time systems and coordination**
* Prioritize **user experience and efficiency**
* Ensure **privacy by design (no real patient data)**
